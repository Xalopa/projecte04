---

# 📘 Breu descripció

## 🧠 Context inicial

En la tasca anterior vam establir les bases de l’administració remota amb:

> 🔐 **SSH (Secure Shell)**

Ara ampliem el concepte cap a entorns amb interfície gràfica.

---

## 🪟 Problema real

En l’entorn professional és habitual trobar situacions com:

- ❌ “No em funciona el programa X”
- ❌ “M’ha sortit un error a la pantalla”
- ❌ Incidències d’usuari final

---

## 👨‍💻 Necessitat tècnica

Com a consultors:

- No només gestionem backend
- També donem suport directe a l’usuari
- Hem de veure i controlar el seu escriptori

---

# 🧰 Tecnologia principal: RDP

## 🖥️ Què és RDP?

> **RDP (Remote Desktop Protocol)** és el protocol d’escriptori remot de Microsoft.

---

## 🎯 Funcionalitats

- 🖱️ Control remot de ratolí i teclat
- 🖥️ Visualització completa de l’escriptori
- 🔧 Suport tècnic en temps real
- ⚙️ Administració de sistemes amb GUI

---

## 🌍 Compatibilitat actual

Tot i ser de Microsoft:

- 🪟 Windows Server i Windows client
- 🐧 Entorns GNU/Linux amb escriptori GNOME
- 🧩 Sistemes com Zorin OS amb suport RDP

# 🛠️ Desenvolupament de la pràctica

## 💻 Entorn de treball

Es faran servir màquines virtuals per simular:

- 🪟 Windows (clients o servidor)
- 🐧 Linux amb entorn gràfic (Zorin / GNOME)

---

## 🔌 Connexió via RDP

### 🪟 Cas Windows

- Activació de connexió remota
- Accés des d’un altre equip client
- Control complet del sistema

---

### 🐧 Cas Linux (GNOME / Zorin)

- Activació del servidor RDP
- Connexió des d’un client remot
- Control de l’escriptori gràfic
