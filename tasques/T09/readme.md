# 📁 T09 · Servidor de fitxers Linux: NFS (tasca individual)

---

# 📘 Breu descripció

## 🧠 Introducció

Benvinguts, equip de consultors júniors.

En aquesta fase del projecte ens trobem amb un requisit molt habitual en entorns professionals Linux:

> 📂 **Centralització de dades en xarxa**

---

# 🏢 Cas client: DevOptimize Solutions

## 👨‍💻 Context

El client és una startup de desenvolupament de programari que treballa exclusivament amb Linux.

---

## ⚠️ Problema detectat

Actualment tenen:

- 📁 Còpies locals de codi font
- 🧾 Documents de disseny duplicats
- ⚙️ Scripts dispersos entre equips

---

## ❌ Conseqüències

Això provoca:

- 🔁 Errors de versions
- 📉 Pèrdua d’eficiència
- 🧠 Desorganització del treball

---

# 🎯 Solució proposada

## 📡 NFS (Network File System)

La solució nativa en entorns Linux és:

> 🧩 **NFS (Network File System)**

---

## ⚙️ Objectiu del sistema

Implementar un sistema que permeti:

- 📁 Compartir directoris entre equips
- 🖥️ Accés centralitzat a fitxers
- 🔐 Control de permisos a nivell de sistema

---

# 🧪 Demostració del sistema

## 🧑‍🏫 Objectiu de la pràctica

Simular un entorn real amb:

- 🖥️ Servidor NFS (NFSv3)
- 💻 Client Linux

---

## 👥 Simulació d’usuaris

Per representar el client real:

- 👤 Creació d’usuaris
- 👥 Creació de grups
- 🔐 Gestió de permisos

Es treballa amb:

- 📄 `/etc/exports`

## 🎯 Objectiu tècnic

Demostrar:
