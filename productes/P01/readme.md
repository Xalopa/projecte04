# 🚀 P01: GitHub - Treballant de forma col·laborativa: Forks i Pull Requests

## 📖 Breu descripció

Ara que dominem les eines de manera individual (servidors, permisos, etc.), és hora d'aprendre a treballar com ho fem realment a **EverPia**: com un equip coordinat.

A la nostra consultora, és impensable que tothom treballi directament sobre una única branca `main` o que s'enviïn fitxers de configuració per correu electrònic. Treballem amb un **flux de treball professional de Git i GitHub**, amb jerarquies definides (Caps d'Equip i Júniors) i un procés estricte de revisió de codi mitjançant **Pull Requests**.

En aquesta activitat simularem el procés d'**Onboarding** (incorporació) de nous consultors, configurant el repositori central de la nostra empresa.

> 🎯 **Objectiu**
>
> El vostre objectiu no és només pujar un fitxer, sinó dominar el flux de treball col·laboratiu de GitHub, incloent:
>
> * Forks
> * Branches
> * Pull Requests
> * Revisió de codi
>
> Aquesta és una de les habilitats més demandades al sector tecnològic.

---

# 🔄 Flux de Treball de GitHub a EverPia

Treballareu en els vostres grups habituals.

Cada grup haurà de designar:

* 👨‍💼 **1 Team Leader (Líder d'Equip)**
* 👨‍💻 **Membres de l'Equip**

El Team Leader serà l'encarregat de coordinar la feina i validar les contribucions dels membres abans de la seva integració al repositori principal.

# ✅ Competències es treballeran

* Utilització de Git i GitHub en entorns professionals.
* Creació i gestió de forks.
* Treball amb branques (*branches*).
* Creació i gestió de Pull Requests.
* Revisió i integració de canvis.
* Treball col·laboratiu en equip.
* Aplicació de bones pràctiques de desenvolupament.
